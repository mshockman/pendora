MenuItem Class

Represents a single item in a menu.

MenuItem

A component that represents a single item in a menu. The item can either be a standalone selectable item that will trigger an action when clicked or it can contain a sub-menu that will open when actived. Sub-menus can be nested as many layers deep as needed.

Config Parameters

config.target : {HTMLElement|String}

The html element to use as the template for the menu item.

Default: null
config.text : {String}

The inner text of the menu item when building from scratch.

Default: null
config.action : {Function}

An action to make when the menuitem is selected.

Default: null
config.href : {String}

Sets the menuitem href attribute on the anchor tag.

Default: null
config.toggle : {"on"|"off"|"none"|"both"}

Controls how the item behaves when clicked.

Takes one of four choices: "on", "off", "none", or "both".

If "on" the menuitem will toggle on when clicked but won't toggle off when clicked.

If "off" the menuitem will toggle off but not on.

If "none" the menuitem will not toggle on or off when clicked.

If "both" the menuitem will toggle on and off when clicked.

Default: inherit
config.autoActivate : {Boolean|Number}

Controls if the item will activate on mouse over if the parent is not active.

If the parent is active the property "openOnHover" is used instead.

Can be either a boolean value or a number. If a number that item will activate after that amount of time in milliseconds has elapsed. If true the item will activate immediately; equivalent to setting autoActivate to 0 milliseconds. If false the item will only activate on mouse over.

Default: inherit
config.openOnHover : {Boolean|Number}

Controls if the item will activate on mouse over if the parent is active.

When the parent isn't active this is controlled by the property "autoActivate".

This property can either be a value or a number greater then or equal to 0. If a number the item will activate after that amount of time in milliseconds has passed. True is equivalent to setting a time of 0 milliseconds. If false or a negative number the item will never activate on mouse over.

Default: inherit
config.delay : {Number}

Adds a delay between item activation and the displaying of it's submenu. Time is given in milliseconds.

Default: inherit
config.closeOnSelect : {Boolean}

If true the item will close if it encounters a select event.

Default: false
config.closeOnBlur : {Boolean}

If true the item will close if the user clicks outside of it.

Default: false
config.classes : {String}

A string of space separated classes to add to the dom element.

Default: null
config.timeout : {Boolean|Number}

The amount of time the item will remain active after the mouse leaves it.

If the value is false or a negative number the item will never timeout.

Default: false
config.nodeName : {String}

The node that is used to render the menuitem.

Default: div
config.position : {Function}

A function that can be supplied to position the item's submenu when the submenu is displayed.

Default: null
config.multiple : {Boolean}

This property can be inherited by the child submenu. If true multiple items can be activate at the same time.

Default: false
...contextAdditional properties that are passed to the render method when not building from a target.

render({text : String, nodeName : String, href : {String|null}) : HTMLElement

@private @overrideable

If not target domElement is provided, this method is called to render the dom elements of the component.

If you subclass this component you can overridden this method to customize the dom.

Config Parameters

text : {String}

The displayed text of the menuitem.
nodeName : {String}

The type of node to use as the root domElement of the menu-item.

Default: div
href : {String|null}

If provided sets the href of the interval anchor element.

Default: null

Usage:

activate(show=true)

Activates the menuitem and display's it's submenu if it has one and show is true.

You can also pass a number greater then or equal to 0 to cause any submenu to display after the specified amount of time in milliseconds.

Config Parameters

show : {Boolean|Number}

Controls if the submenu is displayed when activated.

If true the submenu will be displayed immediately.

If a number the submenu will be displayed after the given amount of time has elapsed in milliseconds.

Default: true

Usage:

deactivate()

Deactivates the menu-item, hiding it's submenu if it has one attached..
