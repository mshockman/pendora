MenuBar

A component for create a horizontal list of pull down items that can display a nestable hierarchy of submenus..

Overview

The goal the the MenuBar class is to create a menubar navigation component that behaves in a similar manner to the menubars that are commonly found in desktop applications. The Menubar component is most commonly initialized via the dom by setting data-init="menubar" and supplying the other necessary marker attributes. Less commonly, you can also create a Menubar by calling the Menubar class.

Positioning

By default a dropdown button displays is menu at it's bottom left corner and will flip it once if the menu would go out of bounds.

CSS3 Animations

As the submenu is show / hidden it will add and remove several state classes.

The first state class is the visible css class. This class is added to the submenu when it is visible.

The second state class is the hidden css class. This class is added to the submenu when it is hidden.

The third state class is the show css class. This class is added when the menu is displayed using the show() method.

The final state class is the hide css class. This class is added when the menu is hidden using the hide() method.

show and hide and intended to be targeted for css3 animations. When an element only have the visible or hidden classes it should display without any animation. So at the start of the page all dropdown menus should have the hidden class and not animations should run. After the user interacts with them the first time the show and hide classes will also be added when the state changes. You should target .visible.show and .hidden.hide to apply css3 animations.
